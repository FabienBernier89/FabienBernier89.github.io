Lexa API (intégration)

Lexa API permet aux équipes Legal Ops et DSI de brancher la traduction juridique de Lexa directement dans leurs propres outils (gestion juridique, gestion documentaire, portails, workflows internes), sans changer les habitudes des utilisateurs. Cette rubrique explique à quoi sert l'API, comment l'intégrer, le principe de clé d'API, les webhooks, les SLA et le support technique. Lexa API relève de l'offre Enterprise : pour la mettre en place, le point de départ reste un échange avec notre équipe via la page Lexa API.

À quoi sert Lexa API et à qui s'adresse-t-elle ?

Lexa API est une API REST qui rend la traduction juridique de Lexa appelable depuis vos propres applications. Au lieu de passer par une interface séparée, vos équipes traduisent là où elles travaillent déjà.

Elle s'adresse surtout aux profils Legal Ops et DSI qui veulent :

  • intégrer la traduction à un DMS (gestion documentaire), un LMS (gestion juridique), un portail client ou un intranet ;
  • automatiser une étape de traduction dans un workflow existant (revue contractuelle, mise en conformité) ;
  • industrialiser le traitement de gros volumes.

Vous accédez au même moteur que le reste de Lexa : 40+ langues, 14 moteurs spécialisés et 900+ lexiques officiels. Pour le détail, consultez la page Lexa API.

Comment intégrer Lexa API à mon DMS ou à mon workflow ?

L'intégration suit une logique simple, en quatre étapes :

  1. Appelez l'API : envoyez une requête avec le texte, la langue cible et le domaine du droit concerné.
  2. Lexa traduit : le moteur juridique spécialisé et les lexiques officiels s'appliquent au contenu soumis.
  3. Recevez l'événement : un webhook signé notifie votre système dès que la traduction est prête, score qualité inclus.
  4. Récupérez le résultat : votre application affiche ou stocke la traduction au bon endroit, sans copier-coller ni export manuel.

Le cadrage de l'intégration (outils, volumes, niveaux de service) se fait avec notre équipe. Demandez une démo depuis la page Lexa API ou écrivez à contact@lexamt.com.

Comment obtenir et utiliser une clé d'API ?

L'authentification se fait par clé d'API : chaque appel transmet votre clé pour identifier et sécuriser la requête.

  1. Échangez avec notre équipe via la page Lexa API pour cadrer votre intégration (l'API relève du plan Enterprise).
  2. Vous recevez vos clés d'API ainsi qu'un environnement de test pour valider vos appels avant la production.
  3. Joignez la clé à chaque requête, dans l'en-tête d'autorisation prévu par la documentation.
  4. Conservez la clé de façon sécurisée, côté serveur : elle ne doit jamais être exposée publiquement.

Le détail des points d'accès et le format des réponses (JSON) sont fournis dans la documentation technique remise lors de l'intégration.

Comment fonctionnent les webhooks pour le traitement par lots ?

Les webhooks évitent d'interroger l'API en boucle. Le principe :

  1. Vous soumettez une ou plusieurs traductions via l'API.
  2. Lexa traite chaque tâche de son côté, en arrière-plan.
  3. Dès qu'un résultat est prêt, Lexa envoie un événement signé à l'URL de webhook que vous avez configurée (identifiant de la tâche, statut, score qualité).
  4. Votre système réagit à l'événement et récupère la traduction correspondante.

C'est la méthode recommandée pour le traitement par lots et l'orchestration de workflows sur de grands volumes. La signature de l'événement vous permet d'en vérifier l'authenticité côté réception.

Quels formats et quelles langues sont disponibles via l'API ?

Via l'API, vous accédez au même moteur juridique que le reste de Lexa :

  • 40+ langues de traduction ;
  • 14 moteurs spécialisés par domaine du droit ;
  • 900+ lexiques officiels pour garantir la terminologie juridique.

Pour la traduction de fichiers au fil de l'eau (Word, PDF, Excel, PowerPoint et autres formats, avec mise en page préservée), vous pouvez aussi compléter avec le module Lexa Documents dans le portail. L'API, elle, est pensée pour l'intégration programmatique à vos systèmes.

Quels SLA et quel niveau de disponibilité propose Lexa API ?

Lexa API est proposée avec des SLA garantis sur la disponibilité et les temps de réponse, ainsi qu'une haute disponibilité de l'infrastructure pour absorber vos volumes critiques.

Les niveaux d'engagement précis sont définis dans le cadre du plan Enterprise, en fonction de vos volumes et de vos contraintes métier. Un NDA est disponible sur demande. Pour cadrer vos besoins, partez de la page Lexa API ou contactez contact@lexamt.com.

Les données envoyées via l'API restent-elles confidentielles ?

Oui. Les garanties de sécurité s'appliquent à chaque appel :

  • chiffrement SSL/TLS en transit et AES-256 au repos ;
  • hébergement exclusivement en Europe, conforme au RGPD ;
  • aucune donnée client utilisée pour l'entraînement ;
  • certifications ISO 27001, AICPA SOC et STAR Cloud Security Alliance ;
  • NDA et SLA disponibles sur demande.

Le détail figure sur la page Sécurité et confidentialité.

Comment gérer la période de rétention des données traitées via l'API ?

La période de rétention détermine combien de temps vos contenus sont conservés avant suppression. Côté compte, elle se règle dans le portail :

  1. Ouvrez votre profil, puis Detail du profil.
  2. Allez dans l'onglet Sécurité.
  3. Ajustez la Période de rétention selon votre politique (par exemple 1 an).

Pour une intégration Enterprise, la politique de rétention et de suppression peut être cadrée plus finement avec notre équipe lors de la mise en place de l'API. Écrivez à contact@lexamt.com pour en discuter.

Combien coûte Lexa API ?

Lexa API relève de l'offre Enterprise, avec une tarification sur mesure selon vos volumes, vos langues et vos niveaux de service. Le plan Enterprise est disponible à partir de 25 utilisateurs.

Pour obtenir une proposition adaptée, demandez une démo depuis la page Lexa API, consultez les tarifs, ou contactez l'équipe commerciale : contact@lexamt.com / +33 1 84 80 21 20.

Comment tester Lexa avant de lancer une intégration API ?

Avant d'engager un projet d'intégration, vous pouvez évaluer la qualité de Lexa en autonomie :

  1. Démarrez l'essai gratuit de 15 jours, sans carte bancaire.
  2. Dans le portail, testez la traduction de texte en temps réel et la traduction de documents sur vos propres contenus.
  3. Validez la terminologie attendue sur vos domaines du droit.

Une fois la qualité confirmée, rapprochez-vous de notre équipe pour cadrer l'intégration API via la page Lexa API.

Quel support technique est fourni pour l'intégration ?

Lexa API s'accompagne d'un support technique dédié, de l'intégration à l'exploitation :

  • documentation des points d'accès et du format JSON ;
  • clés d'API et environnement de test ;
  • un interlocuteur technique pour le suivi du projet.

Pour les questions de mise en place et de cadrage, contactez l'équipe via contact@lexamt.com ou +33 1 84 80 21 20. Pour une question liée à votre compte ou au portail, le support produit reste joignable à support@lexamt.com.

Faut-il choisir entre Lexa API et les autres modules de Lexa ?

Non, les usages sont complémentaires :

  • Lexa API connecte la traduction à vos systèmes pour les flux automatisés et les gros volumes ;
  • Lexa Texte et Lexa Documents servent au travail au fil de l'eau dans le portail ;
  • Lexa Word permet de traduire sans quitter Word ;
  • Lexa Writing aide à reformuler, anonymiser, changer de style ou résumer.

Beaucoup d'organisations combinent l'API pour l'industrialisation et les modules du portail pour les besoins ponctuels. Vue d'ensemble sur la page Toutes les fonctionnalités.
